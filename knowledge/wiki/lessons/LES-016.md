## Aliases

## Owner

## Evidence

- AOS-21 diverse-batch reality test (`docs/PORTFOLIO_PYDANTIC_AI.md`, `.archetype/portfolio/example-voting-app/scan.json`): scanning `dockersamples/example-voting-app` (a polyglot multi-service app) detected the node (`result/package.json`) and python (`vote/requirements.txt`) manifests but **missed `worker/Worker.csproj`** — verified present on disk. The scanner recognizes the **C# language** (from `Program.cs`) but not its **manifest**, so `package_managers=['npm','pip']` omits `nuget`/`dotnet` and the C# service has no ecosystem record. Contrast: `go.mod` **is** detected (`gin` → `package_managers=['go']`), so coverage is python/node/go but not .NET; Java (`pom.xml`/`build.gradle`) and Rust (`Cargo.toml`) are untested and likely also missed.

## Content

- Event: on a polyglot repo, a real service's manifest (`Worker.csproj`) was not detected; the language appears in `language_mix` but has no manifest/`package_manager` entry, so the DNA under-reports the repo's ecosystems.
- Source: AOS-21 diverse-batch reality test, self-found on a real external repo.
- Category: self-found-defect
- Lesson: **manifest detection covers python/node/go but silently drops other ecosystems** — a service whose only manifest is `.csproj` (or `pom.xml`/`build.gradle`/`Cargo.toml`) is invisible to `package_managers`/DNA even though its language is counted. Broaden the manifest table (`.csproj`/`.sln` → dotnet, `pom.xml`/`build.gradle` → jvm, `Cargo.toml` → cargo, `go.sum`, `Gemfile`, `composer.json`, …) so polyglot repos report all ecosystems. Until then, treat `package_managers` as a lower bound, not a complete list.
- Loop feed: scoped follow-up (broaden manifest/ecosystem detection). Open until it lands.
- Status: open — improvement queue. Consumed when manifest coverage is broadened and this row is closed citing it.
