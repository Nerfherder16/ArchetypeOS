# Design Intelligence

## Purpose

Design Intelligence helps choose the visual language, UI patterns, typography, color direction, and interaction model that best fit a product.

It is not trend selection. It is product-specific design reasoning.

## Core Question

What visual language best communicates this product and supports its users?

- cards
- tables
- command palette

- timelines
- steppers
- workflow canvas
- kanban
- graphs
- master detail
- split panes

## Typography And Color

Typography and color should be chosen based on trust, density, accessibility, brand feel, and product domain.

## Product DNA Examples

CPA software should emphasize trust, clarity, accessibility, auditability, and dense workflows.

Media software should emphasize imagery, motion, immersion, and remote-friendly navigation.

Developer tools should emphasize density, keyboard control, dark-mode support, graphs, logs, and command workflows.

## Output

- ranked design directions
- recommended components
- typography direction
- color direction
- accessibility notes
- rejected styles
- rationale
- moodboard references

## Principle

Design recommendations must support product purpose, not fashion.
