# LES-004 — Pin dead resources when tests assert service-down states

## Aliases

## Evidence

- PR #39 Orchestrator review: `test_health_degraded_redis` relied on conftest's `REDIS_URL=redis://localhost:6379/0` being refused — true in CI, false on any machine with a real local Redis (teevee-1). Fixed by pinning port 9999.

## Content

- Event: builder-written test asserted a degraded state via a default service port; Orchestrator review caught the environment dependence before merge.
- Source: PR #39 review remediation.
- Category: review-remediation
- Lesson: a test asserting that a service is DOWN must point at a guaranteed-dead address (reserved port, invalid host), never a default port whose state varies by machine. Extends AOS-LOCAL-001's lesson (tests must not depend on ambient environment) from "up" dependencies to "down" ones.
- Loop feed: conftest convention; checklist item for future service-state tests.
- Status: closed — consumed by the PR #39 conftest fix and this recorded convention.
