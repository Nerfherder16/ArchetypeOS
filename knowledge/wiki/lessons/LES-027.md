# LES-027 — A malformed `ci.yml` fails the whole workflow with 0 jobs / 0 check-runs — and ruff/compileall/compose-config never see it

## Aliases

- GitHub Actions workflow YAML parse error
- inline `run:` scalar with a colon breaks the workflow
- 0 check runs on a fresh PR
- verify ci.yml parses when a builder edits it

## Status

closed

## Owner

Chief Architect / Orchestrator

## Evidence

- AOS-EMBED-002 (PR #73) first CI run: the workflow run for the head SHA came back **`completed / failure` with 0 jobs**, and the PR's check-run list was **empty** (`get_check_runs` → `total_count: 0`). A run that fails with zero jobs is the signature of a **workflow-parse failure** (GitHub can't build the job graph, so nothing runs and nothing reports).
- Root cause: the builder added an "Embedder tests" job with a single-line `run:` step:
  `run: python -c "... else print('ok: no torch')"`. The `: ` inside `ok: no torch` — in an **unquoted** YAML flow scalar (the `run:` value) — is parsed as a mapping-value indicator: `yaml.scanner.ScannerError: mapping values are not allowed here` at line 143. The nested Python/shell quoting is invisible to YAML; only the top-level scalar quoting matters.
- Critically, **none of the Orchestrator's local gates catch this**: `ruff`, `compileall`, `docker compose config`, and the pytest suites all passed — none of them parses `.github/workflows/*.yml`. The failure was only visible by inspecting the *actual* CI run (which reported 0 jobs), not the PR check-runs.

## Content

- Event: a builder's `ci.yml` edit (a new job with an inline `run:` one-liner containing `: `) broke YAML parsing, failing the entire workflow with 0 jobs and 0 PR check-runs; the Orchestrator's standard verification (ruff/compileall/compose/pytest) all passed and missed it.
- Source: AOS-EMBED-002 CI, self-found by inspecting the workflow run when the PR showed 0 check-runs.
- Category: ci-failure.
- Lesson: **whenever a builder edits `.github/workflows/*.yml`, verify it PARSES** — `python -c "import yaml; yaml.safe_load(open('.github/workflows/ci.yml'))"` — as part of independent verification, because ruff/compileall/compose-config never touch workflow YAML. **Prefer a block scalar (`run: |`) for any inline command**, never a single-line `run:` value containing `: ` (a Python/shell string like `print('ok: no msg')`, a URL, a dict literal). And treat **"CI run = failure with 0 jobs / PR shows 0 check-runs" as a workflow-parse smell**, not a flaky/queued run — go read the workflow run, not just the PR checks.
- Loop feed: **consumed** — AOS-EMBED-002: line 143 rewritten as a `run: |` block scalar (and the message de-coloned to `ok - no torch`); `ci.yml` now parses (8 jobs); a defensive grep confirmed no other single-line `run:` scalar carries a `: `. Added a "workflow YAML parses" step to the Orchestrator's independent-verification checklist whenever ci.yml changes.
- Status: closed — cite this row if a workflow run fails with 0 jobs, or when reviewing any builder edit to `.github/workflows/`.
