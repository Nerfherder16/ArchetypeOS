# LES-017 — SECRET_LIKE_FILENAME flags legitimate test-cert fixtures; the signal lacks test-fixture context

## Aliases

- secret-filename false positive
- test cert flagged
- signal precision gap

## Status

open

## Owner

Chief Architect / Orchestrator

## Evidence

- AOS-21 diverse-batch reality test (`.archetype/portfolio/gin/scan.json`): scanning `gin-gonic/gin` raised `SECRET_LIKE_FILENAME` (warning) twice — on `testdata/certificate/key.pem` and `testdata/certificate/cert.pem`, which are **TLS test fixtures**, not real secrets. The heuristic correctly flags credential-shaped filenames but has no notion of test-fixture paths (`testdata/`, `tests/`, `fixtures/`, `examples/`), so it produces noise on well-formed real repos. Echoes LES-001 (credential-shaped strings in test fixtures trip the secret scan).
- **Reinforced at scale** (`.archetype/portfolio/kubernetes/scan.json`): `kubernetes/kubernetes` ships **133** secret-like files (test certs / crypto fixtures) and the scanner emitted **71** `SECRET_LIKE_FILENAME` warnings — the precision gap becomes a **usability** problem at scale (dozens of fixture warnings bury any genuine hit).

## Content

- Event: a scanner risk signal fired on obvious TLS test fixtures, treating `testdata/*.pem` the same as a secret at the repo root.
- Source: AOS-21 diverse-batch reality test, self-found on a real external repo.
- Category: self-found-defect
- Lesson: **`SECRET_LIKE_FILENAME` is working-as-intended but imprecise** — flagging `.pem`/`.key` is defensible (defense in depth), yet without **test-fixture-path awareness** it is noisy on real repos that ship cert fixtures. Refinement: down-rank (or annotate) matches under `testdata/`/`tests/`/`fixtures/`/`examples/` so a genuine root-level `secrets.env` still stands out. Precision, not suppression. Same family as LES-001 (fixtures vs real secrets).
- Loop feed: consumed by AOS-SCAN-PRECISION-001 — the scanner downgrades `SECRET_LIKE_FILENAME` to `severity="info"` (dropping it from `risk_flags`/DNA) when the path is under a test-fixture dir (`testdata`/`tests`/`test`/`fixtures`/`__tests__`/`spec`/`testfixtures`); still emitted, just non-warning. The guardian's separate, overridable secret-block is intentionally unchanged (never-weaken the guardian).
- Status: closed (consumed by AOS-SCAN-PRECISION-001).
