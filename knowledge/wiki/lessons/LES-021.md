# LES-021 — `ClaudeCodeProvider` inherits the ambient project context; a council agent reasoned about the wrong repo

## Aliases

- provider context contamination
- claude -p reads CLAUDE.md / cwd files
- council agent described ArchetypeOS instead of the target

## Status

closed

## Owner

Chief Architect / Orchestrator

## Evidence

- Operator reality test (2026-07-06): run the full ingestion pipeline on `cheahjs/free-llm-api-resources` with the real `claude_code` provider (verdict "Insufficient evidence", conf 0.2875). The review was generated in-session; the real provider is non-deterministic, so the durable record is the verbatim excerpt below rather than a committed artifact.
- The `technology_fitness_judge`'s **evidence array** contained only the target repo's fingerprint — `[{language: Markdown}, {language: YAML}, {language: Python}, {package_manager: pip}]`. Yet its **summary** described **ArchetypeOS itself**: *"a FastAPI API service, a worker, and a scheduler … apps/api (FastAPI 0.115, SQLAlchemy 2.0, Alembic, psycopg3), apps/worker, apps/scheduler …"* and a concern that the scan *"missed the JS/TS web app entirely"* (i.e., ArchetypeOS's `apps/web`). Those version-specific facts can only come from **reading this repository's files** — not from the supplied evidence and not from training.
- Root cause: `ClaudeCodeProvider._build_argv` shells `claude -p <system>\n\n<prompt> --output-format json`, and the subprocess runs in ArchetypeOS's **current working directory**, so `claude` picks up `CLAUDE.md` and has filesystem/tool access to the repo. The agent used it and reasoned about the wrong codebase.

## Content

- Event: a real-provider council agent contaminated its reasoning with the host project's own files, violating the "reason only from the supplied evidence" contract (Evidence over opinion; Verification over inference).
- Source: operator-directed reality test on an external repo, self-found.
- Category: self-found-defect.
- Lesson: **a shelled model provider must be isolated from the ambient project.** `claude -p` inherits the cwd's `CLAUDE.md` and filesystem, so an agent can silently describe whatever repo the process runs in. The council's evidentiary contract requires each agent to reason **only** from the supplied evidence array. Fix: run the provider subprocess in a **clean, empty working directory** (no `CLAUDE.md`, no repo files) and disable ambient tool/file access, so output is a pure function of the prompt. This also protects the future extraction stage (RFC-0008), which will shell a model over real content and must tag provenance rather than absorb ambient context.
- Loop feed: **consumed** — `ClaudeCodeProvider` now runs `claude -p` in a **fresh empty temp directory** (no project `CLAUDE.md`, no repo files) with `--disallowedTools` denying all state-reading/acting tools (Bash/Read/Edit/Write/Glob/Grep/WebFetch/WebSearch/Task/…) and `--strict-mcp-config` ignoring ambient MCP — so the output is a pure function of `system` + `prompt`. A hermetic regression test (`test_claude_code_provider_mocked`) pins the isolated `cwd` + the tool-deny + strict-MCP argv; validated **live** by re-running the free-llm `technology_fitness_judge` prompt, which now reasons only from the supplied evidence ("a lightweight Python project … using pip") with zero ArchetypeOS leakage.
- Status: closed — cite this row if a shelled provider regains ambient context. RFC-0008's real-provider path now has its prerequisite.
