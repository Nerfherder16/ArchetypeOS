# Organizational Intelligence Engine

## Purpose

The Organizational Intelligence Engine reasons across the entire engineering portfolio, not just one repository.

It exists so every repository created with ArchetypeOS can teach every future repository.

## Core Questions

- Have we solved this before?
- Which implementation has worked best across our repositories?
- Which architectural decisions improved maintainability?
- Which mistakes keep recurring?
- Which modules can be reused?
- Which experiments should not be repeated?
- Which benchmarks are relevant to a new project?

## Responsibilities

- Cross-repository recommendations
- Shared pattern detection
- Reuse opportunity detection
- Engineering memory
- Portfolio health scoring
- Architecture comparison
- Benchmark comparison
- Lessons learned extraction
- Duplicate solution detection
- Knowledge transfer between projects

## Inputs

- Repository metadata
- Architecture graphs
- ADRs
- Decision cards
- Research notes
- Benchmarks
- Experiments
- Risk registers
- Lessons learned
- PR Guardian reports
- Release reports

## Outputs

- Portfolio recommendations
- Reuse suggestions
- Shared module candidates
- Repeated-risk alerts
- Migration opportunities
- Portfolio health reports
- Cross-repository timelines

## Example

A new CPA Connector project needs secure document upload. ArchetypeOS detects that another repository already implemented signed URLs, audit logging, tenant isolation, and document retention. It recommends reuse or adaptation with evidence, effort estimate, and migration plan.

## Principle

A project is never isolated. Every project contributes to the shared engineering memory of the portfolio.
