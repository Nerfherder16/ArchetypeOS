## Content

- Event: a plausible-looking but fabricated UUID nearly entered the Board ID Registry — the exact file other sessions trust to update the board without discovery calls. A wrong registry entry fails silently (404s or, worse, updates the wrong item).
- Source: PR #42 preparation, self-caught.
- Category: review-remediation
- Lesson: opaque identifiers (UUIDs, SHAs, run IDs) must be copied from a tool response or fetched from the source system at write time — never reconstructed from memory, however confident the recall feels. If the value isn't in view, fetch it or omit the row.
- Loop feed: registry-update practice; candidate deterministic check for a future guardian evolution (registry rows must match `[0-9a-f-]{36}` AND registry-touching PRs could require a fetch citation).
- Status: closed — consumed by the corrected registry row and this recorded practice.
