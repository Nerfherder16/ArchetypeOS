# Dashboard Interface

## Purpose

The ArchetypeOS dashboard is the visual command center for the engineering portfolio.

It should feel like an operating environment for engineering decisions, not a generic admin panel.

## Primary Areas

- Portfolio Overview
- Project Workspaces
- Architecture Spine Graph
- Voice Inbox
- Research Inbox
- Decision Queue
- Recommendation Queue
- PR Guardian Queue
- Agent Council Results
- Knowledge Graph Explorer
- Reports
- Settings

## Portfolio Overview

Shows every repository and project under management.

Key cards:

- Repository health
- Architecture score
- Security score
- Documentation freshness
- Decision coverage
- Research freshness
- PR Guardian status
- Open risks

## Project Workspace

Each project should include:

- Overview
- Architecture
- Research
- Decisions
- Recommendations
- Risks
- Experiments
- Benchmarks
- Lessons Learned
- Voice Notes
- Reports

## Voice Inbox

Voice Inbox is where captured ideas land before they become actions.

Each item should support:

- play audio
- view transcript
- edit summary
- assign project
- choose intent
- send to agent
- convert to decision
- convert to research task
- convert to issue
- archive

## Visual Direction

The dashboard should support multiple design modes through Design Intelligence.

Candidate styles:

- Enterprise Minimal
- Developer Dark
- Glass Command Center
- Bento Dashboard
- Data Dense
- Architecture Canvas

## Interaction Model

The dashboard should be keyboard friendly, graph friendly, and voice aware.
