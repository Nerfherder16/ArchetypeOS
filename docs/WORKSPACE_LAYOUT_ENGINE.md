# Workspace Layout Engine

## Purpose

The Workspace Layout Engine lets ArchetypeOS adapt its interface to the type of engineering work being performed.

## Workspace Modes

- Architect
- Builder
- Research
- Operations
- Council
- Executive
- Custom

## Architect Workspace

Primary focus:

- Architecture Spine Graph
- Research dossier
- Decision queue
- Recommendation cards

## Builder Workspace

Primary focus:

- Claude Code or local coding session
- terminal output
- PR Guardian
- test results
- implementation tasks

## Research Workspace

Primary focus:

- sources
- source quality
- comparison matrices
- knowledge graph
- research notes

## Operations Workspace

Primary focus:

- container health
- node health
- job queue
- CI status
- cost signals
- local GPU utilization

## Council Workspace

Primary focus:

- agent progress
- agent outputs
- disagreement detection
- Final Judge synthesis

## Executive Workspace

Primary focus:

- portfolio health
- risk summary
- roadmap
- release readiness
- major recommendations

## Multi Monitor Support

Panels should be detachable and remember monitor placement.

Example:

- Monitor 1: code and terminal
- Monitor 2: architecture and graph
- Monitor 3: council and research queue

## Stored Layout Data

Each layout should save:

- active panels
- panel positions
- graph positions
- active project
- active agents
- terminal tabs
- last used filters
