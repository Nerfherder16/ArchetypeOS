# LES-018 — Real Claude JSON output is Markdown-fenced; the tolerant parser must strip the fence

## Aliases

## Status

## Owner

## Evidence

- **First real Agent Council run over external code** (`.archetype/council/pydantic-ai-review.json`, provider `claude_code`, 4 agents, 132s). `claude -p --output-format json` returns the model's text in `.result`; for **3 of the 4 agents** (architecture_cartographer, technology_fitness_judge, security_agent) that text was wrapped in a ` ```json … ``` ` Markdown fence. Only `research_librarian` returned bare JSON.
- `_parse_agent_output` called `json.loads(text)` directly, so the fenced payloads threw and fell into the prose fallback: `confidence=0.05`, `findings=[]`, `concerns=[]`, and the whole fenced block dumped verbatim into `summary`.
- Re-parsing the captured raw summaries with the fix recovers the real content: architecture_cartographer `0→6` findings / `0→8` concerns; technology_fitness_judge `conf 0.05→0.35`, `0→6` findings; security_agent `0→4` findings. The aggregate still abstains (avg ≈ 0.16 < `ABSTAIN_CONFIDENCE` 0.35), so the verdict was unaffected — but the synthesis (agreements / disagreements / unsupported-claims) had been running on empty inputs.

## Content

- Event: a real provider (not the hermetic `DeterministicProvider`) returned structured output the parser silently degraded, because the deterministic test provider never emits a Markdown fence — the gap only exists against the live model.
- Source: first real Council run, self-found on external code (pydantic-ai).
- Category: self-found-defect.
- Lesson: **the single tolerant parser both providers funnel through must be robust to how the live model actually formats JSON** — Markdown code fences and prose preambles are the norm, not the exception. Layered recovery: (1) parse as-is, (2) strip a leading/trailing code fence, (3) slice first `{` … last `}`. Deterministic-provider tests cannot catch this class of defect; a fenced-payload regression test is the guard.
- Loop feed: **consumed in the same change set** — `_loads_tolerant` (fence + prose + brace-slice recovery) in `packages/aos_core/aos_core/services/council.py` plus four `_parse_agent_output` tests in `apps/api/tests/test_council.py` (bare / fenced / prose-then-object / true-prose-fallback).
- Status: closed — cite this row if a provider output-format regression recurs.
