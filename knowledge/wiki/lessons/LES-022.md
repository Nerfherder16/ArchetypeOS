# LES-022 — Distillation's deterministic summary floor emitted badge/analogy noise, and `DNA.frameworks` was never populated — the Transfer Engine was starved

## Aliases

- badge markdown as purpose
- FastAPI analogy mistaken for pydantic-ai's purpose
- frameworks never stamped on DNA
- transfer starved by weak distillation evidence

## Status

closed

## Owner

Chief Architect / Orchestrator

## Evidence

- First **end-to-end reality test** of the intelligence loop (Orchestrator, 2026-07-06): running the real service layer (`run_scan` → `distill_repository` → `recommend_reuse`) over the 6 cloned repos (`free-llm-api-resources`, `example-voting-app`, `gin`, `claude-agent-sdk-python`, `pydantic-ai`, `kubernetes`) proved the loop connects and transfer's ranking is directionally correct, but distillation quality starved it ("right engine, wrong evidence"). Concrete, reproduced failures:
  1. `"container orchestration and scheduling"` → **no matches**, though `kubernetes` is in the portfolio — its `DNA.purpose` was raw **badge markdown** (`[![CII Best Practices]…`), which tokenizes to nothing meaningful, so it scored 0 and was dropped.
  2. `"HTTP routing and middleware"` ranked `gin` **third**, behind `free-llm`/`pydantic-ai` matching on generic `api`/`web` noise — `gin`'s purpose was also badge markdown, and the description lived below a release-announcement heading.
  3. `pydantic-ai`'s purpose was **"FastAPI revolutionized web development…"** — the floor grabbed the README's analogy intro, not what PydanticAI *is* (its description sits in an `<h3>`/`<em>` heading), producing false `web` matches.
  4. **`DNA.frameworks` was empty for all 6** — `run_scan` set `language_mix`/`package_managers`/`runtime_services`/`risk_flags` but never `frameworks`, and the scanner detected package managers from manifest *names* only, never reading bodies for dependencies — so the Transfer Engine's technology-match boost was structurally dead.
- Root cause (verified against source): `extract_repo_knowledge` set `summary = _first_paragraph(lines)` with no cleaning (a line starting with `[` — a badge — passed the prose test), and `run_scan` never populated `dna.frameworks`.

## Content

- Event: the deterministic distillation floor emitted badge/link markup and analogy prose as `RepositoryDNA.purpose`, and the DNA never carried detected frameworks — so the downstream Knowledge Transfer Engine mis-ranked or dropped the very repos a need was about.
- Source: first end-to-end reality test of the intelligence loop, self-found on the real cloned portfolio.
- Category: self-found-defect.
- Lesson: **a deterministic evidence floor must be honest, not noisy — an empty/honest summary is strictly better than badge/analogy noise, because noise actively mis-ranks the retrieval that consumes it.** The floor must drop noise-only lines (badge/image-link runs, bare links, HTML comments, headings) before choosing a summary, prefer the first *declarative description sentence* (`<Name> is/are/provides/… a …`, subject matching the distilled title/repo name — which also skips analogy intros and release-announcement headings), and never emit link-only markup. Separately, a structural field the downstream scorer relies on (`frameworks`) must actually be populated end-to-end: detecting it in the scanner is worthless if `run_scan` never stamps it on the DNA.
- Loop feed: **consumed** — AOS-DISTILL-003 (this package). The summary floor is now a pure, LLM-free `_clean_summary` (badge/heading/comment stripping + declarative-sentence preference + honest fallback; unit-tested on badge-first/analogy-first/all-badge synthetic READMEs). The scanner reads manifest BODIES (`package.json`/`requirements.txt`/`pyproject.toml`/`go.mod`, byte-capped, tolerant) against a curated framework table and emits `frameworks`; `run_scan` stamps `RepositoryDNA.frameworks`. A repeatable regression gate (`scripts/reality_test_distillation.py`, manual — needs the cloned portfolio) re-runs the fixed needs and confirms the fix: **kubernetes now ranks #1 on "container orchestration and scheduling"** (sole match on `container`, from `container-packaged`), **gin ranks #1 on "HTTP routing and middleware for a web API"** (matching `http`/`routing`/`web`/`api`), and pydantic-ai's purpose is the "Pydantic AI is a Python agent framework…" sentence with the FastAPI analogy gone.
- Status: closed — cite this row if a deterministic evidence floor regresses to emitting markup/analogy noise, or if a scanned structural field is computed but never stamped onto the DNA the scorer reads.
