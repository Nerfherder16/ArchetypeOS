# LES-045 — pure type-widening PRs require Guardian overrides with explicit rationale; the seam gap is the defect, not a missing view or test

## Aliases

- contract-lag PR Guardian BLOCK on missing-docs / missing-acceptance-evidence
- type-only api.ts change triggers web-tests-not-enforced
- coherence probe nightly override pattern
- AOS-SELFHEAL-005 guardian override

## Status

validated

## Owner

Nightly coherence routine (AOS-SELFHEAL-005)

## Evidence

- 2026-07-13 coherence run: `tools/coherence_probe.py` detected 10 missing fields across
  `Connector`, `Job`, `Repository` frontend types. Guardian BLOCKed for
  `missing-docs`, `missing-acceptance-evidence`, and `web-tests-not-enforced` on
  the first PR attempt. All three blocks were correctly overridden: the change is
  compile-time only, acceptance evidence is the clean `npm run build`, and docs are
  unnecessary for a type-seam fix whose record lives in the coherence digest.
- Lesson: when the coherence routine widens a type to match an existing backend
  `*Read` schema, the affected files are limited to `apps/web/src/api.ts`. No new
  runtime behaviour is introduced. Guardian's structural checks (docs, e2e tests,
  acceptance sections) are still required but are satisfied by override + rationale
  rather than new files.

## Rules

1. **Always add `## Acceptance Evidence`** — even for type-only PRs. The evidence
   is the clean build exit code, not a test file.
2. **Override `web-tests-not-enforced` when no new UI behaviour is introduced.**
   Type widening is additive and compile-time; new Playwright specs belong in the
   follow-up PR that adds a view consuming the field.
3. **Override `missing-docs` when the coherence digest is the record.** The
   `.archetype/coherence/<date>.md` file is the authoritative description of what
   drifted and why. A separate ADR or feature doc would be redundant.
4. **Every override must cite this lesson (LES-L22) to satisfy
   `override-without-lesson-citation`.** Do not use override tokens without a
   lesson citation — Guardian will BLOCK.
5. **Article XII still applies.** Never widen a type just to silence the probe.
   If a field is genuinely unused, leave it out and document the omission in the
   PR body.
