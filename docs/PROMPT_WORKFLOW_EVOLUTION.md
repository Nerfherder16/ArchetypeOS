# Prompt And Workflow Evolution

## Purpose

ArchetypeOS should version, evaluate, and improve the prompts and workflows that power its agents and engines.

Prompts and workflows are engineering artifacts. They need tests, versioning, evidence, and rollback.

## Managed Artifacts

- Prompts
- CLAUDE.md files
- Agent instructions
- MCP configurations
- Orchestration graphs
- Workflow templates
- Review rubrics
- Final Judge criteria

## Responsibilities

- Track prompt versions
- Compare prompt performance
- Measure acceptance rates
- Detect hallucination patterns
- Detect repeated failures
- Recommend prompt changes
- Recommend workflow changes
- Preserve rollback history

## Evaluation Signals

- recommendation acceptance
- false positives
- false negatives
- user corrections
- task success
- agent disagreement
- time to completion
- output quality
- evidence quality

## Principle

Prompt changes should be treated like code changes. They require rationale, review, validation, and rollback.
