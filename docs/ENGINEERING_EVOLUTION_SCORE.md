# Engineering Evolution Score

## Purpose

The Engineering Evolution Score measures whether a repository is improving over time.

It is not a one-time audit score. It is a longitudinal signal.

## Score Dimensions

- Architecture
- Security
- Documentation
- Testing
- Observability
- Automation
- Knowledge
- Decision quality
- AI readiness
- Reuse
- Maintainability
- Portfolio integration

## Score Behavior

Scores should trend over time.

A repository can add features while its evolution score drops if it accumulates debt, stale documentation, untested paths, unclear decisions, or duplicated implementations.

## Outputs

- Current score
- Trend line
- Delta since last evaluation
- Top score improvements
- Top score regressions
- Recommended corrective actions

## Release Gate

A large score regression should trigger review before release.

## Principle

Progress is not measured only by feature count. It is measured by the system becoming healthier, clearer, safer, and more reusable.
