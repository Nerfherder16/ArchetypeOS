# LES-010 — Moving code across a package boundary must carry its test-enforcement with it

## Aliases

- guardian goes blind to relocated code
- test-enforcement follows the code
- package extraction leaves an enforcement gap

## Status

validated

## Owner

Chief Architect / Orchestrator

## Evidence

- Before AOS-CORE-001, `check_tests_for_code_changes` keyed only on `apps/api/app/`, `apps/worker/app/`, and `apps/web/src/`. Extracting the domain layer to `packages/aos_core/` moved the scanner, models, database, config, and the `scan`/`digest` services out of every path the guardian watched.
- A change to `packages/aos_core/services/scan.py` with no accompanying test change would have passed the test-enforcement check silently — the exact code most in need of the check (the extracted persistence logic) became invisible to it.
- Fix shipped in the same change set: `check_tests_for_code_changes` now sets `core_changed` for `packages/aos_core/` and requires a test change under `packages/aos_core/tests/` or `apps/api/tests/`, else BLOCK `missing-core-tests` (override `PR_GUARDIAN_OVERRIDE_TESTS`). Covered by `test_core_change_requires_tests` and `test_core_change_with_tests_clean`.

## Content

- Event: RFC-0006 Phase 1 relocated the domain core from `apps/api/app/` to `packages/aos_core/`. The PR Guardian's test-enforcement is path-keyed; it did not know about the new location, so the relocated code would no longer trip the "code changed without tests" BLOCK.
- Source: AOS-CORE-001 (self-caught during the extraction).
- Category: process
- Lesson: a package boundary is also an enforcement boundary. When code moves, the deterministic gates that watched it (guardian path checks, CI test targets, coverage globs) do not follow automatically — they must be extended in the same change set, or the guardian goes blind to the moved code precisely when it matters most. Treat "which paths does each gate watch?" as a required checklist item for any code relocation.
- Loop feed: consumed by this package — the guardian's `check_tests_for_code_changes` was extended to cover `packages/aos_core/` with tests, mirroring the api/worker rules. Future extractions into `packages/` should add the corresponding guardian path + a `missing-*-tests` rule as part of the move.
- Status: closed — consumed by AOS-CORE-001: `missing-core-tests` BLOCK added and tested alongside the extraction.
