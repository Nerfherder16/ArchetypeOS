# ArchetypeOS Context

ArchetypeOS is an Engineering Intelligence Platform.

Its mission is to help engineers research, model, decide, build, validate, and evolve software systems with evidence-backed reasoning.

## Core idea

ArchetypeOS sits before and beside code generation. It helps answer what should be built, why it should be built that way, what alternatives exist, and how decisions should evolve over time.

## Core principles

- Research before implementation.
- Evidence over opinion.
- Fitness over familiarity.
- Architecture is a first-class artifact.
- Every important decision is remembered.
- Verification is preferred over inference.
- Human approval is required for destructive actions.
- Local-first operation is preferred when practical.

## Main engines

- Engineering Evaluation Standard
- Research Engine
- Knowledge Graph
- Architecture Studio
- Architecture Spine Graph
- Decision Intelligence
- Technology Fitness Engine
- Recommendation Intelligence
- Design Intelligence
- Build Intelligence
- Evolution Engine
- PR Guardian
- Report Generator

## Runtime vision

ArchetypeOS should run as a local-first control plane, usually on CasaOS or Portainer. It can coordinate worker containers, Claude Code, local LLM nodes, WSL developer nodes, GitHub integrations, and future cloud connectors.

## Knowledge strategy

GitHub stores platform source and documentation. Postgres stores structured runtime data. A graph layer stores relationships. Obsidian can store human-readable research and decision notes. External tools such as Graphify may enrich code and document graphs.
