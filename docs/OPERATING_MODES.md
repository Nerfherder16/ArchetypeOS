# Operating Modes

ArchetypeOS operates in three primary modes.

## 1. Architect Mode

Architect Mode is used before or during implementation.

Responsibilities:

- research options
- challenge assumptions
- create RFCs
- model architecture
- compare technologies
- evaluate tradeoffs
- create decision cards
- produce implementation plans

## 2. Builder Mode

Builder Mode executes validated plans.

Responsibilities:

- coordinate Claude Code
- coordinate local LLMs
- run deterministic tools
- scaffold repositories
- write code when approved
- run tests
- update reports
- create pull requests

## 3. Curator Mode

Curator Mode makes the engineering portfolio smarter.

Responsibilities:

- update documentation
- refresh research
- detect stale architecture
- find better external tools
- detect duplicate implementations
- propose shared modules
- identify recurring risks
- maintain knowledge packs
- update repository DNA
- recommend migrations

## Principle

Architect Mode decides. Builder Mode executes. Curator Mode preserves and improves organizational intelligence.
