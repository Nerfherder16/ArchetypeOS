# LES-012 — local ruff scope must match CI's (`apps/api`, not `apps/api/app`) or migrations lint-slip

## Aliases

- alembic migration lint gap
- narrow-scope ruff false-green
- local-vs-CI lint mismatch

## Status

validated

## Owner

Chief Architect / Orchestrator

## Evidence

- AOS-KNOW-002 (PR #51): the new migration `apps/api/alembic/versions/0004_knowledge_page_nullable_project.py` carried a boilerplate `import sqlalchemy as sa` it never used (the batch-alter uses `aos_core.models.GUID()` for `existing_type`). CI's "API tests and lint" job runs `ruff check apps/api` (the whole tree, incl. `alembic/`) and failed with `F401 sqlalchemy imported but unused` (run 28760110922). The Orchestrator's local pre-push run used `ruff check apps/api/app` — narrower than CI, so it never linted the migration and reported false-green. Fixed by removing the unused import; re-verified with `ruff check apps/api packages/aos_core` (CI scope).

## Content

- Event: a CI lint job failed on a new alembic migration for an unused import that local verification did not catch, because the local ruff invocation scoped to `apps/api/app` while CI scopes to `apps/api` (which includes `apps/api/alembic/`).
- Source: AOS-KNOW-002 CI failure (PR #51 first run), self-diagnosed from the job log.
- Category: ci-failure
- Lesson: **the Orchestrator's independent lint must use the exact CI scope** — `ruff check apps/api packages/aos_core apps/worker tools`, never the narrower `apps/api/app`. Alembic migration files live under `apps/api/alembic/` and are linted by CI; a per-`app` scope makes migration lint errors invisible locally. Also: alembic autogenerated migrations ship a default `import sqlalchemy as sa` that is unused when the migration only does a `batch_alter_table` with a project TypeDecorator (`aos_core.models.GUID()`) — drop the import. Same family as LES-011 (local verification that does not exercise what CI exercises is a silent gap).
- Loop feed: migration fixed in this package; delegation prompts + the Orchestrator's local recipe corrected to lint `apps/api` (full) — candidate guardian/CI-parity check: a pre-PR script that runs the exact CI lint command.
- Status: closed — consumed by the PR #51 migration fix and this recorded scope-parity practice.
