# LES-034 — A lease is not a fence without a token, and declared handler metadata is inert until something reads it at runtime

## Aliases

- claim_token fencing / stale-worker last-writer-wins
- HandlerSpec.timeout_seconds / max_attempts / result_schema were declared but never enforced
- AOS-JOB-FENCING-001 closing lesson

## Status

closed

## Owner

Orchestrator (cloud session)

## Evidence

Verified on `main` @ `267f50f` while implementing AOS-JOB-FENCING-001 (the first package of the post-AOS-REVIEW-002 remediation wave; consumes [[LES-033]]):

- **The lease was not a fence.** `jobs.py` had a correct compare-and-swap `claim_job` (`:151-165`) and a `renew_lease`, but the terminal transitions — `complete_job` (`:193`), `release_for_retry` (`:208`), `fail_job` (`:220`) — did `db.get(Job, job_id)` then mutated with no ownership predicate. So the recovery machinery (lease + reaper) existed, yet a worker whose lease expired and whose job was reclaimed by another worker could still call `complete_job(job_id, …)` and overwrite the newer claim's result. `claimed_by` (a `hostname:pid` string) is not a fence — it can recur, and it was not even checked on completion.
- **`renew_lease` was dead code.** It existed and was exported, but a full-repo grep found zero runtime callers — a handler running past `DEFAULT_LEASE_SECONDS` would silently have its lease reaped mid-flight.
- **Declared handler metadata was inert.** `HandlerSpec.timeout_seconds`, `max_attempts`, and `result_schema` (`registry.py:46-49`) were populated per handler and asserted for shape in a unit test — but the worker's `run_job` read none of them: it called `spec.run` with no timeout wrapper, used a module-level `MAX_ATTEMPTS=3` instead of `spec.max_attempts`, and never validated the result against `result_schema`. The fields read like enforced policy; they were documentation.

## Content

- **Event:** implementing durable-job fencing surfaced that the merged AOS-JOBS-RELIABILITY-001 had the *shape* of fenced execution (leases, a reaper, a renew function, rich `HandlerSpec`) without the *enforcement*: no fencing token on transitions, an uncalled renewer, and three inert metadata fields.
- **Category:** self-found-defect (found while extending the subsystem, verified against source).
- **Lesson (transferable):**
  1. **A lease is only a fence if every state transition proves current ownership with a value the previous owner cannot still hold.** Add a per-claim token (or monotonic generation) minted on each successful claim, and make renew/complete/fail/retry compare-and-swap on it. Reclaiming (by another worker or the reaper) mints a new token / clears the old one, so the stale owner's next write affects 0 rows. Checking `claimed_by` is insufficient — worker ids recur and the transitions didn't even check it.
  2. **A declared field is not enforced behavior until a runtime path reads it.** `timeout_seconds`/`max_attempts`/`result_schema` existing on a dataclass — and being shape-asserted by a test — proves nothing about execution. This is the [[LES-033]] rule ("data/pure-function ≠ enforcement boundary") one layer down: assert the *effect* (a handler that overruns is actually interrupted; a 2-attempt budget dead-letters on attempt 2 not 3; a schema-invalid result is rejected), not the presence of the field.
  3. **Single-threaded workers can enforce a hard timeout with SIGALRM/`setitimer`; a background renewer needs its own DB session and a `finally` stop.** Don't run the handler in a second thread sharing the handler's session (SQLAlchemy sessions are not thread-safe) — interrupt the main thread instead, and give the renewer thread its own session factory (and never name its stop-event `_stop`, which shadows `threading.Thread._stop` and raises `'Event' object is not callable` on `join`).
  4. **Concurrency invariants must be tested on the concurrent database.** SQLite serializes writes, so a "single winner" test there does not exercise the CAS under contention. A race + stale-completion test against PostgreSQL (12 threads through a barrier, real row locks) is the oracle; a green SQLite test is not.
- **Loop feed:** the fencing primitives (Claim token, ownership-guarded transitions) are reused by AOS-NODE-EXECUTION-001 (node-authenticated claim). The "assert the effect not the field" rule feeds the remaining wave packages (authority classification, sensitivity enum) where server-owned enforcement replaces declared-but-unread metadata.
- **Status:** closed — consumed by AOS-JOB-FENCING-001 (branch `claude/aos-job-fencing-001`), which adds `Job.claim_token`, fences all worker-side transitions, wires the renewer, and enforces the three HandlerSpec fields, with PostgreSQL race tests.
