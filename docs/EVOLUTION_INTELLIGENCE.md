# Evolution Intelligence

## Purpose

Evolution Intelligence continuously evaluates how repositories and the full engineering portfolio should change over time.

It does not only ask what is wrong today. It asks what will become risky, obsolete, expensive, duplicated, or strategically limiting over the next 6 to 12 months.

## Core Questions

- What should evolve next?
- What is becoming obsolete?
- What should be replaced?
- What should become a shared library?
- What should be split, merged, deprecated, or retired?
- What technical debt is compounding?
- What future scaling bottlenecks are visible now?

## Responsibilities

- Detect architecture drift
- Recommend framework migrations
- Detect obsolete patterns
- Forecast technical debt
- Predict scaling bottlenecks
- Recommend package consolidation
- Suggest repository splits or merges
- Track AI model evolution
- Recommend new engineering practices
- Produce evolution roadmaps

## Outputs

- Evolution roadmap
- Technical debt forecast
- Architecture drift report
- Technology migration recommendation
- Cost optimization report
- Portfolio modernization plan

## Review Model

Evolution recommendations are strategic. They must go through Decision Intelligence and Final Judge review before implementation.
