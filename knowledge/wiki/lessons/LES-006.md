## Content

- Event: the same guardian warning fired on three consecutive UI packages without ever producing a work item — an alert that never changes behavior is functionally invisible.
- Source: PRs #27, #34, #36.
- Category: guardian-gap
- Lesson: recurring warnings need an escalation path — either they eventually gate (N recurrences → package required) or they are consciously accepted and documented; silent repetition is the worst state.
- Loop feed: web test-coverage package candidate (Alpha guidance #5); recurrence-escalation is a guardian-evolution candidate for AOS-PRG-003.
- Status: open — neither the web tests nor the escalation mechanism exist yet.
