# LES-026 — Shared coordination logs need a union merge driver, not worktrees

## Aliases

## Evidence

- AOS-20 (PR #68) had to re-sync onto `main` three times because the remote session merged PRs #69 and #70 while it was open; each remote merge re-conflicted the same shared coordination logs.
- Behavioral proof of the fix: with `log.md merge=union`, two branches that each prepend a distinct entry merge cleanly (exit 0, both entries retained, 0 conflict markers); the identical merge WITHOUT the union attribute conflicts (exit 1, conflict markers). See `.gitattributes`.

## Content

- Event: two parallel Orchestrator sessions both append to `docs/ACTIVE_WORK.md` and `docs/RECENT_CHANGES.md` on every package. Concurrent additive edits (a new work item, a new changelog entry, a new lesson row) conflict on every merge, so whichever branch merges second must re-sync — the "tandem treadmill."
- Source: AOS-20 tandem run (laptop + remote sessions), 2026-07-06.
- Category: process.
- Lesson: for additive concurrent edits to shared logs, use git's built-in `union` merge driver (`.gitattributes` `merge=union`) — it keeps BOTH sides' lines on conflict instead of emitting markers, and GitHub honors it so the PR stays mergeable. Worktrees do NOT fix this: they isolate working directories, not the merge graph, and the two sessions were already isolated (different branches/environments). Union does not resolve two sides editing the SAME existing line (a status flip); that stays single-writer by protocol (the reconciliation owner flips status). Heavier structural options — per-item fragment files (a `docs/active_work/` dir + `changelog.d/`) or moving the live board to Plane — were considered and deferred behind YAGNI: do the union driver first, escalate only if residual cosmetic conflicts prove painful. Note that moving the board to Plane also trades against the local-first / git-is-source-of-truth constitution and Plane's known flakiness, so it is an RFC-level decision, not a quick fix.
- Loop feed: `.gitattributes` union driver shipped for `ACTIVE_WORK.md`, `RECENT_CHANGES.md`, `lessons/index.md`.
- Status: closed.
